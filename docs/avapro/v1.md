### Slope / Aspect Handling (Recent Adaptation)

AvAPro still uses the classic “virtual slope” idea: each aspect is a separate SNOWPACK file (flatfield pack + virtual slopes), not a re-projection of one profile by compass direction alone.

| Quantity | Role in AvAPro |
|----------|----------------|
| **Aspect** (`SLOPE_ASPECTS`) | Selects **which** SNP `.pro` / `.smet` file to load |
| **Azimuth** (`SlopeAzi` in the `.pro`) | **Unused** by AvAPro |
| **Slope angle** (`SlopeAngle` in the `.pro`) | Used to project layer heights / thicknesses to the vertical (`× cos(angle)`) |
| **`alp` in the ini** (typically `38`) | Fixed angle for **instability / stress** (Sk<sub>38</sub>, etc.) for **all** aspects |

For aspect runs, SNP usually already has ~38° in the file; height correction uses that angle, while stress still uses `alp = 38`.

**Short version:** aspect = which SNP file; azimuth unused; angle used for height projection; instability assumes 38° for all.

Redesign in progress: **[AvaPro v2](v2.md)** (xarray / multi-PAP / full SMET time — physics still being ported).

AvAPro derives daily avalanche problems (new snow, wind slab, persistent, aging-persistent, wet snow) from numerical snow-cover model output. The pipeline is compatible with [SNOWPACK](https://www.slf.ch/en/services-and-products/snowpack.html) `.pro` / `.smet` files. Crocus support is partially prepared via threshold sections (`scmopt = cro`) but is not the primary path.

Background on avalanche problems: [EAWS avalanche problems](https://avalanche.report/education/avalanche-problems).

#### Pipeline Overview

```text
.ini config
    │
    ▼
MAIN_ava_prob_seasons.py
    │
    ├─► match aspects → .pro / .smet pairs
    │
    ├─► find_aps.find_aps()
    │       │  (snowpro reads profiles + meteo)
    │       │  (fu_instab, fu_tau_p, get_ac / get_s)
    │       ▼
    │   df_P_*.pkl , df_met_*.pkl
    │
    └─► post_processing.assigne_avaprobs()
            ▼
        df_P_*_avaprobs_.pkl
```

1. **Config** — paths, season window, aspects, WL and problem thresholds.
2. **Aspect selection** — map `SLOPE_ASPECTS` to SNOWPACK file stems.
3. **`find_aps`** — detect and track potential weak layers; compute instability metrics.
4. **`post_processing`** — assign avalanche-problem flags from thresholds (NAP / PAP / DAP / wind / wet).

Each selected aspect is processed independently in a loop.

#### How to Run

##### 1. Prepare SNOWPACK Output

- For **8 aspects + flat**, set `NUMBER_OF_SLOPES = 9` in the SNOWPACK ini.
- Produce matching `.pro` and `.smet` for each slope.

Two file-naming styles are supported (see `SNP_NAMING` below).

**Append naming** (classic / Rendezvous-style, default):

| Aspect | Suffix | Example stem |
|--------|--------|----------------|
| flat   | `_1`   | `Whistler_Rendezvous_HRDPS_2026_1` |
| N      | `_11`  | `…_11` |
| NE     | `_12`  | `…_12` |
| E      | `_13`  | `…_13` |
| SE     | `_14`  | `…_14` |
| S      | `_15`  | `…_15` |
| SW     | `_16`  | `…_16` |
| W      | `_17`  | `…_17` |
| NW     | `_18`  | `…_18` |

**Insert naming** (NWP station plots / ISSW26 ops): digit inserted in the stem (`Bow_Summit1_…`), not `…_11`:

| Aspect | Example stem |
|--------|----------------|
| flat   | `Bow_Summit_Banff_Bowsumit_HRDPS_2026` |
| N      | `Bow_Summit1_Banff_Bowsumit_HRDPS_2026` |
| NE … NW | `Bow_Summit2_…` … `Bow_Summit8_…` |

**Note:** With the older 5-slope setup (`NUMBER_OF_SLOPES = 5`), aspects were only flat + N/E/S/W as `_1`, `_11`–`_14`. With 9 slopes, **E/S/W indices shift** (`_13/_15/_17`). Do not mix 5-slope files with an 8-aspect AvAPro config.

##### 2. Configure the Ini

Edit (or copy) a file under `config_files/`, e.g. `config_files/2026/NWP_stations/avaprob_Bow_Summit26.ini`.

###### `[avaProbs_general]`

| Key | Role |
|-----|------|
| `SNP_INPUT_FOLDER` | Directory containing `.pro` / `.smet` |
| `SNP_NAMING` | `append` (default) or `insert` |
| `SNP_STATION` | Flat-run basename for **append** naming (optional; see below) |
| `SNP_STATION_BASE` / `SNP_STATION_SUFFIX` | Required for **insert** naming (`BASE` + `1`…`8` + `SUFFIX`) |
| `OUTPUT_NAME` | Output subfolder name under `output_path` (preferred) |
| `output_path` | Output root directory |
| `SLOPE_ASPECTS` | Space-separated list, e.g. `flat N NE E SE S SW W NW` |
| `season_start` / `season_end` | Season window (`YYYY-MM-DD`) |
| `initilization_type` | `station` (AWS / no initial snow profile) or `profile` |
| `scmopt` | `snp` or `cro` — selects which threshold section is used |
| `release` | `trigger` or `natural` |
| `drytime` / `wettime` | Time-of-day windows used in WL / wet logic |
| `rerun_find_WL` | `1` = recompute WLs; `0` = load existing `df_P` / `df_met` pickles |
| `rerun_assign_avaprobs` | `1` = re-assign problems; `0` = load existing `*_avaprobs_.pkl` |

**`SNP_STATION` (append):** required when file stems differ from the input folder name (e.g. folder `Rendezvous/` but files `Whistler_Rendezvous_HRDPS_2026_1.pro`). If omitted, MAIN uses the folder name as the stem (legacy layout such as `RENDEZVOUS1` / `RENDEZVOUS11`).

**Output folder:** `output_path` + `OUTPUT_NAME` (or `SNP_STATION_BASE`, else last component of `SNP_INPUT_FOLDER`).

###### Threshold Sections

- `[avaProbs_thresholds_aps]` — weak-layer detection / tracking.
- `[avaProbs_thresholds_avaprob_snp]` — problem assignment when `scmopt = snp`.
- `[avaProbs_thresholds_avaprob_cro]` — same for `scmopt = cro`.

Comments under `[INFO]` document typical meanings of the thresholds.

#### Aspect Mapping in MAIN

`MAIN_ava_prob_seasons.py` builds file stems from the ini. Two modes:

**`SNP_NAMING = append` (default)** — flat = `SNP_STATION`, aspects append `1`…`8` (→ `…_11`…`…_18` when the flat stem already ends in `_1`):

```text
flat → SNP_STATION
N    → SNP_STATION + "1"   →  …11
NE   → … + "2"             →  …12
…
NW   → … + "8"             →  …18
```

**`SNP_NAMING = insert`** — for NWP station files like `Bow_Summit1_Banff_…`:

```text
flat → BASE + SUFFIX
N    → BASE + "1" + SUFFIX
NE   → BASE + "2" + SUFFIX
…
NW   → BASE + "8" + SUFFIX
```

Supported keys in `SLOPE_ASPECTS`: `flat`, `N`, `NE`, `E`, `SE`, `S`, `SW`, `W`, `NW`.  
You can run a subset (e.g. `SLOPE_ASPECTS= flat N S`) as long as the matching files exist.
