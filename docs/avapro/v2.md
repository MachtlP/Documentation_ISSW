**AvAPro v2** is a redesign of the avalanche-problem pipeline that turns SNOWPACK `.pro` / `.smet` runs (flat + virtual slopes) into a multi-aspect time series of problem types (NAP / PAP / DAP / wind / wet). Sibling / reference implementation: **[AvaPro v1](v1.md)** (`avapro_jul26`).

Background: [EAWS avalanche problems](https://avalanche.report/education/avalanche-problems) · [SNOWPACK](https://www.slf.ch/en/services-and-products/snowpack.html).

## 1. What v2 Is

AvAPro still means **avalanche problems from snowpack models**: rule-based extraction of forecast-style problem types from numerical snow-cover simulations.

| | |
|--|--|
| **Input** | SNOWPACK `.pro` (layered profiles) + `.smet` (meteo / HN24, drift, SWE, …), typically flat + aspects |
| **Output** | An **xarray** product with dimensions **`aspect`** and **`time`**: tracked weak layers, instability metrics (when ported), and selected problem flags |
| **Why** | Automate EAWS-style problem notions for research, climatology, and operational prototyping (e.g. ISSW) |

### Target Pipeline

```text
config.ini
    │
    ▼
avapro.main
    │
    ├─ resolve aspects → file stems (.pro / .smet)
    ├─ read SMET  → ds_met(aspect, time)
    ├─ read PRO   → ds_pro(aspect, time, layer)
    ├─ track_weak_layers  → ds_wl
    ├─ assign_problems    → ds_prob (+ *_sele flags)
    └─ write ds_met.nc, ds_prob.zarr|.nc, optional daily aggregate
```

v2 is currently a **redesign scaffold**: architecture, config, aspect dimension, and control flow are in place; the **full physics from v1 is not fully ported** yet. You can configure aspects, load SMETs, and run end-to-end with placeholders — scientifically valid problem series need PRO reading + layer search + instability ported from v1.

## 2. Changes vs v1 (AvaPro:Jul26)

<p class="table-caption"><strong>Table 1.</strong> Design rules that differ from v1.</p>

| Topic | v1 (`avapro_jul26`) | v2 |
|-------|---------------------|----|
| Time | Sample at `drytime` + `wettime` → ~1 row/day | **All SMET steps** (`time_mode=from_smet`) |
| Aspects | Separate pickles / loop | **`aspect` dimension** in one dataset |
| PAP count | Single PAP; new PAP demotes old → DAP | **Many PAPs**; demote by **depth ≥ 0.90 m** only |
| Wet ISO | Calendar **days** (`dysisomax` ~ 3) | **Hours**, window **`dysisomax_hours=24`** |
| Storage | pandas `df_P` + pickle | xarray → zarr / netcdf |
| Assignment | Separate `post_processing` | Same logic; callable as 2nd pass |

### Other v2 Rules (Target Behaviour)

- **PAP → DAP:** move when slab depth ≥ **0.90 m** (verticalized); new PAPs do **not** auto-demote older PAPs.
- **Wet selection:** consecutive isothermal hours ≤ **24 h** (not multi-day calendar ISO).
- **Instability gating:** `instab_policy = every_step | on_change | every_Nhours` (needed because time is hourly).
- **Naming:** still supports `append` (`…_1` / `…_11`…) and `insert` (`Bow_Summit1_…`) like v1 MAIN.

### Porting Map (High Level)

| v1 | v2 target |
|----|-----------|
| `MAIN_ava_prob_seasons.py` | `avapro/main.py` |
| `find_aps.py` | `avapro/core/find_wl.py` |
| `post_processing.py` | `avapro/core/assign.py` |
| `fu_instab` + helpers | `avapro/core/instab.py` |
| `snowpro/` | `avapro/io/pro.py` |
| `df_P_*.pkl` | `ds_wl` / `ds_prob` zarr/netcdf |
